# Definition Skills

## Purpose

Definition skills establish clarity before execution.

These skills ensure that work is based on:

- explicit expectations
- known consumers
- defined constraints
- structured outputs

Without this layer, all downstream work operates on assumptions.

## When to Use

Use these skills:

- at the start of any task
- when requirements are unclear
- when ambiguity exists
- before making decisions
- before executing work

These are always applied first.

## Skills

### L1.S1 — Define Contract

Defines the explicit expectations between the producer and consumer of the work.

### L1.S2 — Identify Consumers

Identifies all entities that depend on the output, including downstream impacts.

### L1.S3 — Apply Quality Questions

Uses the six-question framework (Outcome, Consumers, Constraints, Structure, Failure Modes, Verification) to fully define the work.

### L1.S4 — Define Constraints

Establishes what must be true and what cannot be violated.

### L1.S5 — Define Structure

Defines what the correct output should look like before building it.

## Usage Guidance

- Always begin work with these skills.
- Do not move to execution until:
  - outcome is clear
  - consumers are known
  - constraints are defined
  - structure is established
- If ambiguity exists, return to this layer instead of pushing forward blindly.

## Failure Signals

- Work starts without clear expectations
- Output is debated after delivery
- Consumers interpret results differently
- Rework occurs due to misunderstanding
