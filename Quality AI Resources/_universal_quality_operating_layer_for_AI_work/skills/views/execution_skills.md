# Execution Skills

## Purpose

Execution skills enable the system to perform work while maintaining alignment with defined expectations and constraints.

These skills transform defined intent into actual output.

## When to Use

Use these skills:

- after definition is complete
- when producing any output
- during writing, coding, planning, analysis, or transformation
- when working under bounded uncertainty

## Skills

### L3.S1 — Execute Work

Performs the task based on the defined contract and constraints.

### L3.S2 — Maintain Contract Alignment

Ensures output remains consistent with expectations during execution.

### L3.S3 — Apply Structured Decision Model

Uses WHY / WHAT / WHERE / HOW to guide decisions during execution.

### L3.S4 — Generate Output

Produces the final artifact (document, code, plan, etc.).

### L3.S5 — Surface Assumptions

Makes all assumptions visible when operating under uncertainty.

### L3.S6 — Handle Ambiguity

Allows progress to continue when uncertainty is bounded without blocking unnecessarily.

## Usage Guidance

- Only execute after sufficient definition exists.
- Maintain continuous alignment with:
  - contract
  - constraints
  - structure
- Make assumptions explicit — never hide them.
- Continue progress when risk is bounded.

## Failure Signals

- Output drifts from expectations
- Assumptions are hidden
- Work proceeds without alignment checks
- Ambiguity is ignored instead of managed
