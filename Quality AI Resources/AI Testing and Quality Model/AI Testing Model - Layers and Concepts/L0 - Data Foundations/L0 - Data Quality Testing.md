# L0 — Data Quality Testing

## What
Data Quality Testing validates that data is usable, well-formed, and fit for purpose.

---

## Why
Poor quality data leads to unreliable model behavior and incorrect system outcomes.

---

## Where
- Training datasets
- Retrieval content
- Input payloads

---

## When
- During data ingestion
- During transformation
- During dataset curation

---

## How
- Data profiling
- Rule-based validation
- Outlier detection

---

## Failure Modes
- Missing or malformed values
- Incorrect formats
- Invalid data types

---

## Verification / Signals
- % valid records
- Data validation rule pass rate
