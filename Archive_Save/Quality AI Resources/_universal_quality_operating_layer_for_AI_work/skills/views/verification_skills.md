# Verification Skills

## Purpose

Verification skills ensure that outputs are correct, testable, and aligned to expectations.

These skills define and execute validation of work.

## When to Use

Use these skills:

- before execution (define verification)
- during execution (align validation)
- after execution (validate correctness)
- when diagnosing issues
- when managing risk

## Skills

### L2.S1 — Identify Failure Modes

Defines how work can fail before it does.

### L2.S2 — Bound Risk

Ensures all uncertainty is resolved or explicitly bounded.

### L2.S3 — Define Verification

Defines how output correctness will be proven.

### L2.S4 — Classify Uncertainty

Distinguishes between resolved, bounded, and unbounded risk.

### L2.S5 — Determine Execution Readiness

Determines whether work should proceed, proceed with caveats, or pause.

### L2.S6 — Derive Test Strategy

Translates risk and verification into structured testing decisions.

### L4.S1 — Diagnose Failure

Identifies root cause based on expectation mismatch.

### L4.S2 — Trace Failure to Contract

Connects failure back to contract definition issues.

### L4.S3 — Validate Output Quality

Ensures output meets defined expectations.

### L4.S4 — Verify Output

Executes validation methods to confirm correctness.

### L4.S5 — Capture Learnings

Improves future work through recorded insights.

### L4.S6 — Support Human Override

Allows human decisions to override system gating when needed.

## Usage Guidance

- Define verification before execution whenever possible.
- Treat failure as a contract mismatch, not just a defect.
- Use risk to determine how much validation is needed.
- Ensure every output can be proven correct.

## Failure Signals

- Outputs cannot be verified
- Validation is improvised or inconsistent
- Failures are treated as isolated issues
- Risk is not connected to testing effort
