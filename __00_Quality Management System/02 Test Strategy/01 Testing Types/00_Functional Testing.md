# Functional Testing

## Purpose

Functional Testing verifies that a solution behaves according to defined business, consumer, technical, and operational expectations.

The objective of Functional Testing is to determine whether the solution performs the functions it was intended to perform and produces the expected outcomes when used under expected conditions.

Functional Testing focuses on validating behavior rather than implementation.

---

## Key Question

> Does the solution do what it is expected to do?

---

## What Functional Testing Validates

Functional Testing may validate:

- Business rules
- Calculations
- User interactions
- Data creation and updates
- Workflow behavior
- System responses
- Error handling
- Data validation
- Authorization rules
- Notifications and messaging
- Reporting outputs

---

## Common Examples

Examples include:

- A login process successfully authenticates a valid user.
- A mortgage calculation returns the correct payment amount.
- A customer record saves successfully.
- A required field generates an error when left blank.
- A user receives an expected confirmation notification.

---

## Common Defects Identified

Functional Testing often identifies:

- Missing requirements
- Incorrect business rules
- Calculation errors
- Workflow failures
- Validation issues
- Data handling issues
- User interface defects
- Requirement interpretation problems

---

## L0-L5 Alignment

Functional Testing can occur at multiple testing levels.

| Testing Level | Typical Usage |
|--------------|--------------|
| L0 | Requirements and acceptance criteria reviews |
| L1 | Unit and component validation |
| L2 | Feature and application validation |
| L3 | Integration validation |
| L4 | Business process validation |
| L5 | Production smoke validation |

Most Functional Testing occurs at L2 and L4.

---

## Automation Considerations

Functional Testing is frequently automated when:

- Validation is repetitive
- Business rules are stable
- Regression protection is desired
- Fast feedback is valuable

Automation should be used strategically to improve confidence and maintainability.

---

## Risks Reduced

Functional Testing helps reduce:

- Incorrect system behavior
- Business rule failures
- User experience issues
- Data integrity concerns
- Release confidence concerns

---

## Success Criteria

Functional Testing is successful when:

- Expected behavior is clearly defined
- Critical business functions are validated
- Significant business risks are addressed
- Sufficient confidence exists that expectations have been met

Functional Testing is one component of a complete testing strategy and should not be used as the sole indicator of quality.
