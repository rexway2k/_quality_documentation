# Testing Model (L0-L5)

## Purpose

This Testing Model establishes common testing levels used across solution delivery.

Each level provides different forms of quality feedback and confidence.

Higher levels are not replacements for lower levels.

Each level serves a unique purpose.

The goal is to provide the right feedback at the right time using the most appropriate validation approach.

---

# L0 — Review and Analysis

## Purpose

Prevent defects before implementation.

Activities may include:

- Requirements reviews
- Acceptance criteria reviews
- Architecture reviews
- Design reviews
- Threat modeling
- Risk assessments
- Peer reviews

### Key Question

> Are we building the right thing?

---

# L1 — Unit and Component Validation

## Purpose

Verify that individual components function correctly.

Activities may include:

- Unit testing
- Component testing
- Static code analysis

### Key Question

> Did we build the component correctly?

---

# L2 — System Validation

## Purpose

Verify functionality within the application or system boundary.

Activities may include:

- API testing
- UI testing
- Functional testing
- Feature validation

### Key Question

> Does the feature work correctly?

---

# L3 — Integration Validation

## Purpose

Verify interactions between systems.

Activities may include:

- API integrations
- Event processing
- Data flow validation
- Contract testing

### Key Question

> Do the systems work together?

---

# L4 — End-to-End Validation

## Purpose

Verify complete business workflows.

Activities may include:

- End-to-End testing
- User journey validation
- Business process validation

### Key Question

> Does the business process work?

---

# L5 — Production Confidence

## Purpose

Verify quality in real-world operation.

Activities may include:

- Monitoring
- Observability
- Synthetic testing
- Production smoke testing
- Operational validation

### Key Question

> Is the solution successfully fulfilling expectations in production?

---

## Important Principles

Earlier feedback is generally:

- Faster
- Less expensive
- Easier to resolve

Higher levels provide:

- Broader confidence
- More realistic validation
- Slower feedback
- Greater execution cost

Effective testing strategies leverage multiple testing levels rather than relying on a single validation approach.

---

## Summary

The L0-L5 Model provides a structured approach for understanding:

- Where testing occurs
- What level of confidence is being provided
- How feedback supports quality decisions

Each level contributes unique value to an overall testing strategy.
