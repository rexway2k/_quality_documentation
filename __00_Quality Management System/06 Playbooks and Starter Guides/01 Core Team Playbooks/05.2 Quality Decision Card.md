# Quality Decision Card

## Purpose

This card is a lightweight companion for day-to-day quality decisions. It is intended for use before test planning, review, estimation, or release discussions.

## Use This Card Before

- test planning
- backlog refinement
- change review
- release readiness discussion
- estimation or sizing conversations

## 1) Define the Contract

Clarify:
- what outcome must be true
- who depends on it
- what constraints must hold
- what can fail
- how the team will prove it works

If any answer is unclear, return to the contract.

## 2) Make the Five Decisions

### Why
What decision are we making? What risk are we trying to reduce?

### What
What kind of risk matters most?

### Where
Where should confidence be built?

### How
How should we validate?

### When
When must this risk be reduced?

## 3) Check for Clarity

Avoid using these as interchangeable labels:
- intent is not level
- attribute is not intent
- level is not execution mode
- execution mode is not the thing being tested
- timing is not a test type

## 4) Define “Enough”

Enough is reached when:
- critical risks are identified
- failure modes are understood
- verification shows those risks are controlled

## 5) Final Check

Can the plan be expressed as:

Intent | Attribute | Level | Execution Mode | Timing

If not, the plan is not clear enough yet.
