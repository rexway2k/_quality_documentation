# Playbooks and Starter Guides

This folder contains practical, lightweight guides designed to make the broader quality, testing, automation, and AI quality content easier to use in real team work.

## Intended Use

These materials are intended to be:
- short
- practical
- easy to scan
- action-oriented
- connected to the broader repository content

## Folder Structure

- 00 Start Here
  - Entry-point guides for how to use the library, the fastest path for new team members, and quick guidance by situation
- 01 Core Team Playbooks
  - Guides for everyday delivery teams and new team members
- 02 Role-Based Starter Guides
  - Short guides tailored to different team roles
- 03 Specialized Playbooks
  - Guides for specific scenarios such as AI quality, automation, and release readiness
  - Recent additions include root cause analysis, defect management, test case management, test automation, and metrics/reporting guidance
- 04 Templates and Checklists
  - Reusable templates and one-page aids
- 05 Source Material References
  - Indexes or pointers to the deeper source content in the main repository
- 06 Working Drafts
  - Drafts and in-progress content

## Content Principles

Each playbook or starter guide should:
- explain when and why it should be used
- stay short and digestible
- provide simple steps or questions to ask
- include a short section on source materials for deeper reference
