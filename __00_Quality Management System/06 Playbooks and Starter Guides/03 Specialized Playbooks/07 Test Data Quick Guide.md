# Test Data Quick Guide

## Purpose

This guide helps teams think about test data in a simple and practical way.

## When to Use This Guide

Use this when planning validation, preparing test scenarios, or working around limited or sensitive data.

## Core Idea

Good test data supports realistic validation. It should be sufficient to exercise important paths and edge cases without becoming overly complex.

## Practical Guidance

- use data that reflects the real scenario where possible
- include boundary and failure cases when meaningful
- protect sensitive information and comply with policy
- make sure the data supports the quality concern being tested

## Quick Checklist

- [ ] The data supports the scenario being tested
- [ ] Key edge cases are represented where needed
- [ ] Sensitive data is handled appropriately
- [ ] The test data is enough to make the validation useful
