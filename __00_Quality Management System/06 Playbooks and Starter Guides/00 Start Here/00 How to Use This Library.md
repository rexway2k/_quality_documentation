# How to Use This Library

## Purpose

This library is designed to help teams find the right guidance quickly. The goal is to reduce the need to read everything before doing useful work.

## Recommended Mental Model

Use the library in this order:

1. Start with the most relevant entry point
2. Use a playbook or guide for the immediate task
3. Open the deeper source material when more context is needed

## Best Starting Points

### If you are new to the quality model
Start with:
- [Team Quality 101 Playbook](../01 Core Team Playbooks/01 Team Quality 101 Playbook.md)
- [What to Test, Where, When, and How Playbook](../01 Core Team Playbooks/02 What to Test, Where, When, and How Playbook.md)

### If you need to make a decision today
Start with:
- [Quality Decision Playbook](../01 Core Team Playbooks/05 Quality Decision Playbook.md)
- [Release Readiness Playbook](../01 Core Team Playbooks/04 Release Readiness Playbook.md)
- [Defect Triage and Root Cause Playbook](../01 Core Team Playbooks/06 Defect Triage and Root Cause Playbook.md)

### If you are working in a specific role
Use the relevant starter guide:
- [Product Owner Starter Guide](../02 Role-Based Starter Guides/01 Product Owner Starter Guide.md)
- [Engineer Starter Guide](../02 Role-Based Starter Guides/02 Engineer Starter Guide.md)
- [QA/QE Starter Guide](../02 Role-Based Starter Guides/03 QA QE Starter Guide.md)
- [Support and Operations Starter Guide](../02 Role-Based Starter Guides/04 Support and Operations Starter Guide.md)
- [Leadership and Delivery Manager Starter Guide](../02 Role-Based Starter Guides/05 Leadership and Delivery Manager Starter Guide.md)

## When to Use the Deeper Sources

Open the standards and reference material when you need:
- a policy or enterprise expectation
- more detail on a specific topic
- a stronger basis for a release or governance discussion
