# Fastest Path for New Team Members

## Purpose

This guide provides the shortest useful path for someone who wants to become productive quickly with the quality library.

## Recommended Reading Path

1. Read [Team Quality 101 Playbook](../01 Core Team Playbooks/01 Team Quality 101 Playbook.md)
   - understand the basic quality mindset and team expectations

2. Read [What to Test, Where, When, and How Playbook](../01 Core Team Playbooks/02 What to Test, Where, When, and How Playbook.md)
   - learn how to think about validation in practical terms

3. Read [Story and Backlog Quality Playbook](../01 Core Team Playbooks/03 Story and Backlog Quality Playbook.md)
   - learn how quality starts during planning and refinement

4. Read [Release Readiness Playbook](../01 Core Team Playbooks/04 Release Readiness Playbook.md)
   - understand what good release readiness looks like

5. Read your role-based starter guide
   - [Product Owner Starter Guide](../02 Role-Based Starter Guides/01 Product Owner Starter Guide.md)
   - [Engineer Starter Guide](../02 Role-Based Starter Guides/02 Engineer Starter Guide.md)
   - [QA/QE Starter Guide](../02 Role-Based Starter Guides/03 QA QE Starter Guide.md)
   - [Support and Operations Starter Guide](../02 Role-Based Starter Guides/04 Support and Operations Starter Guide.md)
   - [Leadership and Delivery Manager Starter Guide](../02 Role-Based Starter Guides/05 Leadership and Delivery Manager Starter Guide.md)

## What This Path Gives You

- a basic understanding of quality in delivery
- a practical way to think about testing and validation
- an overview of common team decisions and release expectations
