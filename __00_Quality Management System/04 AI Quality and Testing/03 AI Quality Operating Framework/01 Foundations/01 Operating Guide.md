# Operating Guide

## Purpose

This guide explains how the AI Quality Operating Framework is used in practice.

## Core Operating Flow

All work follows a consistent flow:
1. define the contract
2. clarify the task using quality questions
3. bound uncertainty
4. define verification
5. execute the work
6. validate the output
7. diagnose and improve if needed

## Key Concepts

- contract definition
- uncertainty management
- verification planning
- traceability and explainability

## Usage Notes

The framework should be applied in a way that is:
- tool-agnostic
- domain-agnostic
- behavior-driven
- composable
- explainable
