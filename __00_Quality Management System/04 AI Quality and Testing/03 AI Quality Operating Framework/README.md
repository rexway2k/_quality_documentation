# AI Quality Operating Framework

## Purpose

This section captures a reusable quality operating framework for AI-enabled work. It is intended to help teams apply a consistent quality model to AI agents, AI-assisted workflows, and human-AI collaboration.

## What This Framework Covers

This framework brings together:
- the core quality concepts that apply to AI work
- a structured operating model for defining contracts, risks, and verification
- a skill-based view of how AI quality work can be organized and applied
- practical implementation guidance for building and testing AI-enabled behavior

## Suggested Use

Use this section when you want to:
- understand how quality applies to AI work
- define expectations and contracts for AI output
- bound uncertainty and define verification
- structure AI agent behavior in a repeatable way
- evaluate or improve AI-enabled workflows

## Structure of This Section

- 01 Foundations
  - system specification and operating guidance for the framework
- 02 Skills
  - skill taxonomy, skill library, prompt blocks, schemas, and skill views
- 03 Verification
  - evaluation rubric, test scenarios, and validation guidance
- 04 Implementation
  - agent build guidance, implementation notes, and supporting scripts

## Relationship to the Rest of the Repository

This section complements the broader AI quality and testing content by providing an operational model for how quality can be applied in practice.
